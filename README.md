#### Висновки

__Жадібний алгоритм__ для суми 10 000 виконується майже миттєво (наприклад, ~0.00001 сек), оскільки його складність $O(k)$, де $k$ — кількість номіналів монет.

__Динамічне програмування__ для тієї ж суми працює значно довше (наприклад, ~0.1 сек), бо його складність $O(k \cdot n)$, де $n$ — сума.

Підсумок:
для великих сум і стандартних номіналів монет найбільш ефективним є __жадібний алгоритм__ — він у десятки тисяч разів швидший за динамічне програмування.
Жадібний алгоритм швидший, бо виконує набагато менше операцій — лише по одному для кожного номіналу, тоді як динамічне програмування аналізує всі можливі варіанти для кожної суми.
